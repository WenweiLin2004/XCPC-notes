# 2024ICPC香港E
[链接](httpsqoj.accontest1885problem9919)
![在这里插入图片描述](httpsi-blog.csdnimg.cndirect2e5dcfa208ff4980969a43bfa91a64ef.png)
题意：

凹包题

定义凹包为所有的内角中恰好只有一个大于$pi$的内角，并且包含所有给定的$n$个点的集合

计算所有凹包的面积和2

思路：

 首先计算出$n$个点的凸包，那么显然凹进去的点不能是凸包的点
 对于每个凸包内的点作极角排序，那么凹进去的两条边必定是相邻两个点与旋转点的连线
 剩余的部分还必须要满足凸包的要求
 下面通过一个例子来看看所求的面积是哪部分(果然还是得画图，光想就想了一天)

![在这里插入图片描述](httpsi-blog.csdnimg.cndirect8c126617ea17431bbf20fc9e1f63594d.png)
假设我们此时按照$F$点旋转，逆时针排序，为$(A,G,E,H,I,B,C,D)$的顺序

然后我们选择$(A,G)$对凹进去，面积为所有带颜色的区域，如下
![在这里插入图片描述](httpsi-blog.csdnimg.cndirect368d191e610f4010bf9fc1d856fb5e99.png)
继续遍历，选择$(G,E)$对
![在这里插入图片描述](httpsi-blog.csdnimg.cndirectc0d1f35f07204dae81adc858959e68a3.png)
继续遍历$(E,H)$对，因为共线所以直接忽略(PS题目保证没有共线，此处为了仅仅举例)

遍历$(H,I)$
![在这里插入图片描述](httpsi-blog.csdnimg.cndirect799c5f2097714e9faa04e7d6cc4e95c6.png)
遍历$(I,B)$
![在这里插入图片描述](httpsi-blog.csdnimg.cndirect000543a0773b4866ae99fb383b5fb77b.png)
 可以发现，对于一片区域$(1,0,0,0,...,0,1)$，即左边和右边都是凸包的点，中间部分非凸包的点，这一部分可以一起计算，每个凹包的面积就是左边的凸包面积(绿色)+右边的凸包面积(红色)+(凸包面积-大三角形面积)(蓝色)

# 2023ICPC济南M
![在这里插入图片描述](httpsi-blog.csdnimg.cndirect919ba9a200b8473bb9c3ceccb8dd0a14.png)

假设能包含所有点的多边形顶点数最少为$R$,那么计算能包含所有点的多边形，顶点数为$R$或$R+1$

思路：

 显然最少顶点数就是凸包
 枚举凸包内的点，极角排序，如果相邻两个点均为凸包的点，那么答案+1

# 2024CCPC哈尔滨B
![在这里插入图片描述](httpsi-blog.csdnimg.cndirectec61c501fd1147cbb7ab5db9bfae070d.png)
题意：

求最大面积的凹包

思路：

 如果凸包内没有任何点，无解
 当前多边形每选择一个凸包内的点，面积必定会减少；因此最多只会选择一个点凹进去
 一个显然的结论是，这个点必定是内部点形成的凸包的点
 枚举外凸包的每一条边，找到与该边最近的点，减去这个三角形的面积即为凹包的面积
 即找到内凸包的一个点
 类似旋转卡壳的思想，当前凸包每旋转到下一条边，内凸包离边最近的点也会跟着旋转，因此双指针维护即可
 复杂度$O(n log n)$，瓶颈为排序

# 2020ICPC南京I
![在这里插入图片描述](httpsi-blog.csdnimg.cndirect2d8d3362e6c74f239fd94ea9085d484f.png)
 想法很简单，但是对于计算几何苦手来说有点难实现
 实际上$v_x$的范围影响到的是速度斜率的大小，斜率$k$的范围为$frac{-v_y}{v_x} leq k  leq frac{v_y}{v_x}$，斜率改变即改变点的移动方向
 首先观察到$n$很小，最多只有$2n$个点，且不会走回头路，因此按$y$排序，考虑dp，$dp_i$表示到当前点的最小速率
 因为想要过掉一条线段，如果不能直走通过，那么肯定是刚好擦边线段两边的点过最好，因此我们然后枚举更上层的点$j$,判断当前两点的线段是否会和所有线段相交(特判在端点相交，此时算两条边不交)，如果没有与任何线段相交，那么可以到达，计算斜率$k$,$dp_j=min(dp_j,max(dp_i,k))$
 然后对于每个点，判断是否可以直线向上通过区域而不与其他线段相交，如果可以，用当前的点的值更新答案，对答案取最小值
 唯一的坑点是，如果你想要走的线路和障碍重合，那么这条线路也是合法的
 复杂度$O(n^3)$
 官方题解是二分$v_x$，然后判断能不能走到终点

```cpp
struct Point{
    int x,y;
};
int cross(Point a,Point b){
    return a.xb.y-a.yb.x;
}
void solve(){ 
    int n;
    cinn;
    vectorPoint p(n);
    for(int i=0;in;i++)cinp[i].xp[i].y;
    sort(p.begin(),p.end(),[&](auto a,auto b){
        if((a.x0)!=(b.x0))return a.xb.x;
        return cross(a,b)0;
    });
    int ans=0;
    for(int i=0,j=0;in;i++){
        while((j+1)%n!=i&&cross(p[i],p[(j+1)%n])=0)j=(j+1)%n;
        if(j=i)ans=max(ans,j-i+1);
        else ans=max(ans,n+j-i+1);
    }
    coutn-ans'n';
}
```
